Project Snapshot

System Overview

This project builds a low-cost, intelligent mobile robot capable of autonomously detecting and avoiding obstacles using ultrasonic sensing. It also supports Bluetooth and voice command control, providing flexible manual operation when needed.

The system demonstrates the integration of embedded systems, sensors, and wireless communication — making it a compact and interactive robotic solution ideal for educational purposes.

Aim & Objectives

Project Goals

Primary Aim

Design and build an Arduino-based robotic car that automatically detects and avoids obstacles, controllable via Bluetooth and voice commands.

Technical Objectives

Ultrasonic Sensing

Implement HC-SR04 ultrasonic sensing for reliable distance measurement in the 2–400 cm range.

Motor Control

Integrate L293D motor driver for bidirectional control of 4× DC gear motors with current protection.

Bluetooth Interface

Enable HC-05 Bluetooth module for manual remote control via mobile app, up to ~10 m range.

Voice Control

Mobile app interface built in MIT App Inventor with Android speech-to-text API integration.

Learning Goals

Embedded Programming

Hands-on experience with Arduino IDE, serial debugging, and firmware development.

Mobile App Development

Build custom control UI using MIT App Inventor with voice-capable interface.

Low-Cost Automation

Prototype a robust, affordable automation demo suitable for lab and student projects.

Software & App Stack

Software Architecture

Autonomous Obstacle Avoidance

The ultrasonic sensor continuously measures distance. When an obstacle is detected within threshold, the Arduino evaluates position and chooses a deflection direction.

// Autonomous control loop int distance = measureDistance(); if (distance < 30) {   stopMotors();   int left = scanAngle(-45);   int right = scanAngle(+45);   (left > right) ? turnLeft() : turnRight(); } else { moveForward(); }

Bluetooth Remote Control

HC-05 communicates with the MIT App Inventor mobile app using a simple single-letter command protocol over SPP serial.

// BT command receiver char cmd = Serial.read(); switch(cmd) {   case 'F': moveForward(); break;   case 'B': moveBackward(); break;   case 'L': turnLeft(); break;   case 'R': turnRight(); break;   case 'S': stopMotors(); break; }

Voice Command Control

Android's built-in speech-to-text API converts voice input to text on the mobile app, which is then mapped to commands sent to the robot via Bluetooth.

// MIT App Inventor block logic // Voice → Text → BT Command if (voiceText == "forward") → send 'F' if (voiceText == "backward") → send 'B' if (voiceText == "left") → send 'L' if (voiceText == "right") → send 'R' if (voiceText == "stop") → send 'S'

Advantages
  • Autonomous navigation without human intervention
  • Wireless and voice controls add versatility
  • Low-cost components — ideal for labs and student projects
  • Compact, portable and battery-powered
Limitations
  • Voice mode sensitive to ambient noise
  • Bluetooth requires pairing and line-of-sight for best range
  • Single ultrasonic sensor — limited field of view without servo
Next Steps
  • Improve speech robustness with noise filtering
  • Add obstacle mapping (SLAM-lite)
  • Upgrade power management system
  • Modularize code for reuse and scaling
Main Conclusions
Multi-Modal Control

The project successfully demonstrated multi-modal robot control — autonomous, Bluetooth, and voice — seamlessly integrated in a single platform.

Cost-Effective Platform

Arduino-based approach proved cost-effective and accessible for educational purposes. Integration of autonomous and manual control modes was achieved successfully.

Future-Ready Base

Voice control adds user convenience but needs noise filtering improvements. The system is suitable for educational demonstrations and further research development.

Literature Survey

Key Research Insights

🔊

Sensor Choices

IR and ultrasonic sensors are common for proximity detection. Ultrasonic offers reliable range (2–400 cm) and operates across all lighting conditions — preferred for this build.

🌐

Connectivity

Many hobby robots use Bluetooth for manual control; fewer combine it with autonomous behaviour in a single platform — this project bridges that gap.

🎙️

Voice Control

Speech interfaces add convenience but can suffer in noisy environments. Server-side STT processing or robust mobile APIs (Android) improve accuracy significantly.
